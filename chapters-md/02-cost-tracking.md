# Chapter 02: Cost Tracking

The cost tracking subsystem monitors token consumption and financial expenditure throughout a session. It maintains a running tally of the following quantities:

- **Input tokens**: the number of tokens sent to the API across all calls in the session
- **Output tokens**: the number of tokens generated by the model across all calls
- **Cache read tokens**: tokens served from the prompt cache rather than reprocessed
- **Cache write tokens**: tokens written into the prompt cache for future reuse
- **Total cost in US dollars**: the cumulative monetary cost of all API calls
- **API call count**: the number of distinct API requests made during the session

Each time an API call completes, the usage statistics from that call are added to the session totals. The cost in dollars is computed separately (using the pricing information described in Chapter 33) and added to the running total.

When displaying the cost to the user, the system uses two decimal places if the total is one dollar or more (for example, "$3.42"), and four decimal places if the total is less than one dollar (for example, "$0.0037"). This ensures that small costs during brief sessions are still visible and meaningful to the user.
